[ht::head {Wiki Text}] <%nochildlist%> The RAMADDA Wiki facility generally follows the format that Wikipedia follows. For example: [wiki::text { ==Section Heading 1== ===Section Heading 2=== }] A horizontal line: [wiki::text { ---- }] Lists: [wiki::text { * bullet 1 * bullet 2 ** sub bullet 1 ** sub bullet 2 * bullet 3 }] See the Wikipedia page on Wiki Markup for more information on basic Wiki markup. Note: RAMADDA does not support the Wikipedia HTML table format. For tables its best just to use HTML table markup. Linking to other entries Normally in a wiki a link to another page is of the form:
\[\[Page name\]\]
In RAMADDA you can use the name of another entry to create the link. The Entry needs to be a sibling entry to the Wiki Page (i.e., it shares the same parent of the Wiki Page entry) or it needs to be the identifier of some other entry, e.g.,
\[\[7ca13e52-8efb-4d17-863f-e17027dbcc22|Earthquake data\]\]
If the entry with the given name does not exist then the link will take one to the create entry form.

You can add links to other entries with the "Add Link" menu. RAMADDA WIKI Properties You can also import content from the current Wiki Page entry or any other entry with the import syntax:

{{property}}
e.g., in the above example we are importing the Information tab block for the Wiki Page entry with:
{{information}}
For example, if we want to import the information block for the above data file with the block being closed we would do:
{{information entry="7ca13e52-8efb-4d17-863f-e17027dbcc22" open="false"}}
The set of import tags include:
    information name description layout properties html map mapentry comments breadcrumbs toolbar image menu recent gallery tabs grid tree links information name description layout properties html map mapentry comments breadcrumbs toolbar image menu recent gallery tabs grid tree links
  • name The name of the entry.
  • description The description of the entry.
  • comments The comments for the entry.
  • toolbar The icon toolbar.
  • breadcrumbs The breadcrumb path.
  • information The properties information block.
  • image If the entry is an image then the image.
  • links The table of views and actions.
  • children The list of children entries. (see below).
  • subfolders Just the children folders.
  • subentries Just the children non folders.
You can also import any of the views of an entry, e.g., the map view, the opendap view, the calendar view, etc. In the above example we are importing the map view of the earthquake data:
{{import entry="7ca13e52-8efb-4d17-863f-e17027dbcc22" data.point.map width="400"}}
Right now you need to manually enter the output view identifier in the import statement. Look at the URL of a page and you will see "output=identifier". In this case we use html and the import tags to come up with the page layout.

The children tag is used to display the children entries of a folder. It can take the form: 

{{children title="Links" showtoggle="false"}}
The title attribute is used in place of "Children". If showtoggle="false" then the show/hide toggle link is not shown. You can specifiy another entry: 
{{children entry="some other entry id"}}

Other tags include:

Display a map: 

{{map}}
Display a map just for a group: 
{{mapentry}}
If you used the {{map}} tag with a folder then the children get mapped. The mapentry tag says to make a map for just the entry. [wiki::tag recent]

Show the most recent 3 days in a list organized by day: 

{{recent days="3"}}
Show a vertical listing of the children entry images: 
{{gallery level="1"}}
Show a vertical listing of the grand children entry images: 
{{gallery level="2"}}
Set the image width and only show up to 5 images: 
{{gallery level="2" width="300" count="5"}}
Show the children as tabs: 
{{tabs}}
Show the children in a grid: 
{{grid}}
Show the grandchildren in a grid: 
{{grid level="2"}}
Show all of the menu actions: 
{{menu}}
You can specify which menus to show with a comma separated list of menu names: 
{{menu menus="file,edit,feeds,view,other"}}
This only shows the file and view menu: 
{{menu menus="file,view"}}
Show the menu actions in a popup: 
{{menu popup="true" title="Click here"}}
