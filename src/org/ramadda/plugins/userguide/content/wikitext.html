[ht::head {Wiki Text}] <%nochildlist%> There is always some repository entry associated with wiki text. This might be a specific Wiki Entry, we might be treating the description of some arbitrary entry as wiki text using the <wiki> tag, or the wiki text might be coming from a Page Style. Wiki Editor In any of these cases RAMADDA provides a wiki text editor: [ht::cimg images/wikiedit.png {Wiki Text Editor} {border=0 width=600}]
  • The buttons allow you to add basic wiki markup.
  • The Add property menu adds RAMADDA specific properties from the entry that is being displayed (e.g., name, description, children entries, etc).
  • The Add entry id menu allows you to specify the entry to use in properties (see below).
  • The Add entry link menu adds wiki links to other entries.
Basic Markup The RAMADDA Wiki facility generally follows the format that Wikipedia follows. For example: [wiki::text { ==Section Heading 1== ===Section Heading 2=== }] A horizontal line: [wiki::text { ---- }] Lists: [wiki::text { * bullet 1 * bullet 2 ** sub bullet 1 ** sub bullet 2 * bullet 3 }] See the Wikipedia page on Wiki Markup for more information on basic Wiki markup. Note: RAMADDA does not support the Wikipedia HTML table format. For tables its best just to use HTML table markup. Linking to other entries Normally in a wiki a link to another page is of the form: [wiki::text { [[Page name]] }] In RAMADDA you can use the name of another entry to create the link. You can use the Add entry link menu to select an entry. This produces wiki text of the form: [wiki::text { [[entry id|Label to show for link]] }] e.g.: [wiki::text { [[7ca13e52-8efb-4d17-863f-e17027dbcc22|Earthquake data]] }] You can also just specify an entry by name. Either an absolute path: [wiki::text { [[Top entry/Some other entry]] }] Or relative to the wiki entry: [wiki::text { [[Some entry]] }] If the entry with the given name does not exist then the link will take one to the create entry form. Wiki Properties You can also import content from the current Wiki Page entry or any other entry with the import syntax: [wiki::text [wiki::tag property]] e.g., if we wanted to show the information tabs for an entry we'd do: [wiki::text [wiki::tag information]] For example, if we want to import the information block with it open: [wiki::text [wiki::tag information {open="true"}]] This would give us: [ht::cimg images/wikiinformation.png {} {border=0}] Normally, all property tags refer to the relevant entry. However, you can specify another entry: [wiki::text [wiki::tag information {entry="some other entry id or name or 'parent' or 'grandparent'"}]] The entry value can be the ID of an entry, its name (or full path) or "parent" or "grandparent" to denote the parent entry (or its parent) of the current entry. The Add entry id menu in the editor toolbar allows you to select the entry to use. As a convenience this adds both the ID of the entry as well as its name. Basic Wiki Properties The first set of property tags show some information for the entry. Note: the attributes are all optional.
  • [wiki::tagdef name]: Entry name
  • [wiki::tagdef description]: Entry description
  • [wiki::tagdef fromdate {format="date time format"} ], [wiki::tagdef todate {format="date time format"} ]: From date/to date of entry. If specified, format the date with the given date time format. This follows the Java date format conventions.
  • [wiki::tagdef information]: The information tabs.
  • [wiki::tagdef properties]: Show the properties listing.
  • [wiki::tagdef html]: Shows the default html view of the entry. Note: this requires an entry="..." to specify a different entry.
  • [wiki::tagdef map {width="width of map" height="height of map"}]: Display a map. If the entry is a folder then display the children entries of the folder.
  • [wiki::tagdef mapentry {width="width of map" height="height of map"}]: Display a map just for a group. If you used the map tag with a folder then the children get mapped. The mapentry tag says to make a map for just the entry.
  • [wiki::tagdef earth {width="width of map" height="height of map"}]: If enabled, display the Google Earth plugin. To enable the Google Earth see the Admin help.
  • [wiki::tagdef comments]: Show the comments.
  • [wiki::tagdef breadcrumbs]: Show the breadcrumb path.
  • [wiki::tagdef layout]: This shows the "Layout:" toolbar bar for an entry.
  • [wiki::tagdef toolbar]: Show the entry toolbar.
  • [wiki::tagdef image {src="image source" width="image width" link="true|false" left="absolute position" top="absolute position"}]: Include an image. If src is not given then this shows the image (if there is one) from the entry. If src is specified it can take on the form: 
    src="entry name"
    to show an entry. This is equivalent to using entry="...". You can also specify the name of a file attachment with: 
    src="::attachment name"
    Or any attachment with: 
    src="::*"
  • [wiki::tagdef menu {popup="true|false" title="title to use" menus="file,edit,feeds,view,other"}]: Show the menu actions. If menus is not specified show all of the menus. If menus is specified only show those menus listed. Show all menus in a toggle block: 
    {{menu}}
    You can specify which menus to show with a comma separated list of menu names. Only show the file and view menu: 
    {{menu menus="file,view"}}
    Show the menu actions in a popup: 
    {{menu popup="true" title="Click here"}}
Collection Wiki Properties The following properties display a collection of entries. By default they are the children entries of the wiki entry but you can specify other groups of entries with: [wiki::text {{{property entry="some other entry" folders="true" files="true" type="entries of a specific type" level="1 or 2"}}}] If folders is set then only show folder like entries.
If files is set then only show non folders.
If type is set then only show entries of that type.
The attribute level can be set to "1" (the default) or "2". Level 1 refers to the immediate children entries of the given entry. Level 2 refers to the grandchildren entries (i.e., the children of the children).
  • [wiki::tagdef tree]: The [wiki::tagdef tree] tag is used to display the children entries of a folder. It can take the form: [wiki::text [wiki::tagdef tree {title="Links" showtoggle="false" open="true"}]] The title attribute is used in place of "Children". If showtoggle="false" then the show/hide toggle link is not shown.
  • [wiki::tagdef grid]: Show the children in a grid. e.g., show the grandchildren as a grid: [wiki::text {{{grid level="2"}}}]
  • [wiki::tagdef recent {days="number of days to show"}]: Show the most recent N days in a list organized by day.
  • [wiki::tagdef gallery {width="image width" count="max number to show"}]: Show a vertical listing of the children entry images, e.g. [wiki::text {{{gallery}}}] Show a vertical listing of the grandchildren entry images: [wiki::text {{{gallery level="2"}}}] Set the image width and only show up to 5 images: [wiki::text {{{gallery width="300" count="5"}}}]
  • [wiki::tagdef tabs]: Show the children as tabs.
  • [wiki::tagdef links {separator="separator text to use" style="inline css style for the link" class="css class for the link" tagopen="html to prepend to the link" tagclose="close html"}]: Layout links to the entries.
    This produces a set of links to the entries. You can create a header with wiki text like: [wiki::text { <center> {{links separator="&nbsp;|&nbsp"}} </center> }] If you wanted to make a vertical list do: [wiki::text { {{links tagopen="<div>" tagclose="</div>"}} }]
Importing any page You can also import any of the views of an entry, e.g., the map view, the opendap view, the calendar view, etc., with the [wiki::tag import] tag:
[wiki::tagdef import {output=""}]

So, if you wanted to show the calendar view of an entry you would do: [wiki::text [wiki::tag import {output="calendar.calendar"}]] To find the correct output identifier just navigate to the page you want to include and look at the url for the output=... url argument and use that. [ht::foot]
