[ht::head {LiDAR Data Formats}] <%nochildlist%> NLAS/RAMADDA provides support for a number of different LiDAR file formats.
  • LAS - Version 1.2 and 1.3 - all point formats
    Note: for now the Java LAS library does not read the GEOTIFF coordinate reference system information so it is assume that the x/y/z coordinates are geographic - i.e., they are longitude/latitude/meters.
  • GLAS - GLAS record formats 5, 6, 12, 13, 14, 15 and GLANL
    Note: the tiling won't work for the "non-flat" formats - everything except GLANL.
  • LVIS - Version 1.2 and 1.3 - all points formats (LCE, LGE and LGW)
  • ATM, McORDS, PARIS text formats
  • General CSV text formats - See below
CSV Format The NLAS tools include a text file reader. This can read comma, tab and space separated data files. It depends on a properties file to describe the delimiter, coordinate reference system and the fields in the file. For a given data file <data file>.csv the tools look for: 
     /files/point.properties
     /files/<data file>.properties
The tools always look for a point.properties file and then <data file>.properties file.

Here is an example point.properties file for an example data.csv file. The properties file contains: 

########################################################################
## Delimiter
########################################################################
delimiter=,

########################################################################
## Coordinate reference system
########################################################################
crs=geographic

#We also support wgs84
#crs=wgs84

#And utm
#crs=utm

#if utm then also specify the zone and whether its north:
utm.zone=17
utm.north=true

########################################################################
## Fields
########################################################################

#Comma separated list of field names. You must specify the xyz fields with certain names:
#Longitude:
#x or longitude or long or lon
#Latitude:
#y or latitude or lat
#Altitude:
#altitude or elevation or elev or alt

fields=latitude\[unit=degrees\],longitude\[unit=degrees\],altitude\[unit=m\],temperature\[unit=celsius\]

#The "\[...\]" is optional but allows you to specify attributes of the field, e.g., unit.
#Right now we don't do things like missing value, scale factor, etc, but we'll add that
#as needed
